## 一、背景与目标

### 1.1 背景
2026-03-05 已完成工作流自动化工具初步选型，重点评估了 n8n、Make、Zapier 三款主流工具。本文档在该基础上进一步深化，从**场景梳理、工具详细对比、工作流设计、实施路径**四个维度，形成可直接落地的自动化实施方案。

### 1.2 目标
- 梳理日常工作中的重复性任务，明确可自动化的场景
- 提供三款工具的详细功能/定价对比，支撑选型决策
- 设计 5 个高价值自动化工作流，覆盖飞书集成、数据备份、任务提醒等场景
- 制定分阶段落地计划，明确优先级与里程碑

### 1.3 核心假设
- 运行环境：个人/小团队场景，以飞书为主要协作平台
- 数据合规要求：优先考虑国内可访问、数据不出境的方案
- 技术能力：具备一定的命令行和服务器操作能力

## 二、日常工作重复性任务梳理

### 2.1 任务分类矩阵

| 类别 | 具体任务 | 频率 | 耗时/次 | 年化耗时 |
|------|---------|------|--------|---------|
| **信息收集** | 定时爬取行业资讯/竞品动态 | 每日 | 30min | 180h |
| **信息收集** | 汇总多渠道数据到飞书多维表格 | 每日 | 20min | 120h |

| **任务管理** | 任务到期提醒（人工盯盘） | 每日 | 10min | 60h |
| **文件处理** | 文件跨平台同步（云盘/本地/邮件） | 多次/日 | 15min | 90h |
| **数据备份** | 手动备份重要文件到多个位置 | 每周 | 30min | 26h |

## 三、工具详细对比

### 3.1 功能维度对比

| 功能维度 | n8n | Make | Zapier |
|---------|-----|------|--------|
| **部署方式** | SaaS / 自托管 / Docker | 仅 SaaS | 仅 SaaS |
| **开源** | ✅ Apache 2.0 | ❌ | ❌ |
| **免费计划** | ✅ 社区版无限用（自托管） | ❌ 免费版限制 1000 次操作/月 | ✅ 100 任务/月，限两步 |
| **可视化编辑** | ✅ | ✅✅（最强） | ✅ |
| **代码执行** | ✅（内置 Code 节点） | ⚠️ 需付费 | ⚠️ 需付费 |
| **AI 集成** | ✅ LLM节点丰富 | ✅ AI 模块 | ✅ Zapier Copilot |
| **飞书集成** | ✅ 社区节点 + REST API | ⚠️ 无官方节点 | ⚠️ 无官方节点 |
| **自定义节点** | ✅ 可开发 JS/TS 节点 | ❌ | ❌ |
| **执行日志/监控** | ✅ | ✅ | ✅ |

| **并发执行** | ✅ | ✅ | ✅（企业版） |

| **Webhook** | ✅ 无限 | ✅ 有限制 | ✅ 有限制 |
| **数据存储** | ❌ 无内置（需接外部） | ❌ 无内置 | ✅ Zapier Tables（免费版有限） |

#### n8n

| 计划 | 价格 | 执行次数 | 重要限制 |
|------|------|---------|---------|
| **社区版（自托管）** | 免费 | 无限 | 无商业支持，需自行运维 |
| **Starter（云）** | €20/月 | 5,000 次/月 | 仅单用户，无团队协作 |
| **Pro（云）** | €60/月 | 25,000 次/月 | 3 个用户，10 个工作流 |

> 💡 **推荐策略**：有技术能力者首选**社区版自托管**（Docker 一键部署），零成本且数据完全自主。有出海需求或无服务器运维能力者选**Starter/Pro**。

#### Make（原 Integromat）

| 计划 | 价格（年付） | 操作次数 | 重要限制 |
|------|------------|---------|---------|
| **Free** | 免费 | 1,000 ops/月 | 仅单工作流，数据传输 1GB |
| **Core** | $59/月 | 10,000 ops/月 | 无限工作流，数据传输 10GB |
| **Pro** | $129/月 | 50,000 ops/月 | 无限工作流，优先支持 |
| **teams** | $249/月 | 100,000 ops/月 | 5 用户，团队功能 |
| **Enterprise** | 定制 | 无限 | SSO、审计、SLA |

> 💡 **推荐策略**：可视化流程编排能力最强，适合需要复杂条件分支、并联/串联逻辑的场景。性价比高于 Zapier。

#### Zapier

| 计划 | 价格（年付） | 任务数 | 重要限制 |
|------|------------|-------|---------|
| **Free** | 免费 | 100 任务/月 | 仅两步，Zapier Tables 不限 |
| **Professional** | $19.99/月 | 2,000 任务/月 | 无限步骤，Webhooks |
| **Team** | $69/月 | 10,000 任务/月 | 25 用户，共享工作流 |
| **Enterprise** | 定制 | 1M+ | 无限用户，SSO、监控 |

> 💡 **推荐策略**：生态最广（7000+集成），但飞书支持较弱。**飞书相关工作流不推荐使用 Zapier**。

### 3.3 飞书集成能力专项对比

| 飞书功能 | n8n | Make | Zapier | 飞书原生自动化 |
|---------|-----|------|--------|--------------|
| 飞书消息发送 | ✅（HTTP Request） | ✅（HTTP） | ✅（HTTP） | ✅ |
| 飞书机器人 | ✅ | ✅ | ✅ | ✅ |
| 飞书多维表格 | ✅（API） | ✅（API） | ✅（API） | ✅ |
| 飞书审批 | ✅ | ✅ | ✅ | ✅ |
| 飞书日历 | ✅ | ✅ | ⚠️ | ✅ |
| 飞书云文档 | ✅ | ✅ | ⚠️ | ⚠️ |
| 官方认证集成 | ❌ 无官方 | ❌ 无官方 | ❌ 无官方 | ✅ 官方 |

> ⚠️ **重要提示**：三款工具均无飞书官方认证集成节点，均通过 **飞书开放平台 API**（HTTP Request）实现连接。这意味着有一定开发能力的团队可以借助这些工具实现几乎所有飞书功能的自动化。

### 3.4 选型结论

```
┌─────────────────────────────────────────────────────────┐
│                    工具选型建议                           │
├─────────────────────────────────────────────────────────┤
│ 飞书核心工作流自动化  →  飞书原生自动化 + n8n（自托管）      │
│ 复杂跨平台工作流      →  n8n（自托管）或 Make              │
│ 快速原型验证          →  Make（可视化强，上手快）            │
│ 出海业务/海外生态     →  Zapier + n8n（海外节点）           │
│ 数据安全优先          →  n8n（社区版自托管），数据完全自主    │
└─────────────────────────────────────────────────────────┘
```

**最终推荐**：
- **首选方案**：n8n 社区版（自托管）+ 飞书原生自动化
- **备选方案**：Make（复杂跨平台工作流）
- **验证阶段**：先用飞书原生自动化验证核心场景，再迁移到 n8n

## 四、高价值工作流设计

### 4.1 工作流一：飞书任务到期智能提醒

**价值等级**：⭐⭐⭐⭐⭐（P0，最高优先）  
**预计节省**：60h/年

**触发条件**：飞书多维表格中任务「截止日期 ≤ 当前时间 + 24h」且「状态 ≠ 已完成」

**工作流设计**：
```
[飞书多维表格 - 任务表]
       │
       ▼ （定时检查：每小时）
[Filter: 是否到期未完成？]
       │
       ├── 否 → 结束
       │
       └── 是
           │
           ▼
[AI 判断优先级：紧急/重要/一般]
           │
           ▼
[飞书机器人 → 对应群聊]
  - 紧急：@ 责任人 + 加急标记
  - 重要：普通提醒
  - 一般：汇总日报提醒
           │
           ▼
[更新多维表格「已提醒」字段]
```

**实现方式**：n8n（自托管）+ 飞书开放平台 API + Cron

---

### 4.2 工作流二：每日行业资讯自动采集与推送

**价值等级**：⭐⭐⭐⭐（P0）  
**预计节省**：180h/年

**触发条件**：每日 08:30 自动执行

**工作流设计**：
```
[Cron: 每日 08:30]
       │
       ▼
[并行采集多个信息源]
  ├── 36氪 RSS / API
  ├── 虎嗅 RSS
  ├── 微信公众号精选（通过第三方聚合）
  └── 自定义关键词搜索（Google Alerts / 百度指数）
           │
           ▼
[AI 摘要处理]
  - 去重 + 关键词提取
  - 生成 200 字摘要 + 原文链接
           │
           ▼
[飞书机器人 → 资讯推送群]
  格式：「📰 今日资讯 | [日期]」
  内容：标题 + 摘要 + 来源 + 链接
           │
           ▼
[存档至飞书多维表格「资讯库」]
```

**实现方式**：n8n（自托管）+ RSS 节点 + OpenAI/DeepSeek API + 飞书消息 API

---

### 4.3 工作流三：飞书多维表格数据自动汇总报告

**价值等级**：⭐⭐⭐⭐（P1）  
**预计节省**：39h/年

**触发条件**：每周五 18:00 或每月最后一天

**工作流设计**：
```
[Cron: 每周五 18:00]
       │
       ▼
[查询飞书多维表格]
  - 本周新增任务数
  - 已完成任务数
  - 超期任务数
  - 各成员任务完成率
           │
           ▼
[生成汇总报告（模板渲染）]
           │
           ▼
[飞书机器人 → 管理群]
  - 本周任务周报
  - 下周待办预览
           │
           ▼
[发送至负责人飞书消息]
           │
           ▼
[存档至飞书云文档「周报」目录]
```

**实现方式**：n8n + 飞书多维表格 API + 飞书云文档 API

---

### 4.4 工作流四：重要文件自动备份（3-2-1 策略）

**价值等级**：⭐⭐⭐（P1）  
**预计节省**：26h/年

**触发条件**：每周日 20:00 + 文件变更时实时同步

**工作流设计**：
```
[文件变更监控]
  └── 指定文件夹（桌面/文档/项目目录）
           │
           ▼
[并行备份至 3 个目标]
  ├── 目标1: 本地 NAS / 移动硬盘（rsync）
  ├── 目标2: 阿里云 OSS / 腾讯云 COS（国内高速）
  └── 目标3: 加密压缩包 + 版本控制
           │
           ▼
[备份完成通知]
  └── 飞书机器人 → 发送备份报告
           │
           ▼
[多维表格记录备份日志]
  - 备份时间
  - 文件数量
  - 总大小
  - 状态
```

**实现方式**：n8n（自托管）+ S3/OSS 节点 + rsync + 飞书消息 API

---

### 4.5 工作流五：会议自动化全流程

**价值等级**：⭐⭐⭐（P2）  
**预计节省**：40h/年

**触发条件**：飞书日历创建会议时触发

**工作流设计**：
```
[飞书日历 API - 会议创建事件]
           │
           ▼
[提取会议信息]
  - 会议主题、时间、参与者
           │
           ▼
[并行执行]
  ├── [生成会议议程模板]
  ├── [查询参与者日历冲突]
  └── [准备往期会议记录摘要]
           │
           ▼
[发送会议准备通知]
  ├── 主持人：议程草稿 + 参会提醒
  └── 参会人：会议议程 + 相关资料
           │
           ▼
[会议结束后]
  ├── [自动生成会议纪要模板]
  └── [发送满意度调查]
```

**实现方式**：n8n + 飞书日历 API + AI 摘要 + 飞书消息 API

```
   └─────────────┘          └─────────────┘          └─────────────┘
```

### 7.2 数据流说明

1. **入站数据**：飞书事件（任务变更、会议创建等）通过 Webhook → n8n
2. **定时任务**：n8n Cron 节点触发信息采集、数据汇总
3. **出站数据**：处理结果通过飞书机器人/API 回写
4. **备份数据**：文件变更 → n8n → 云存储（OSS）+ 本地（NAS）

### 7.3 安全考量

- **飞书 API 凭证**：存储在 n8n 环境变量，不硬编码
- **数据不出境**：所有数据处理在境内服务器完成
- **备份加密**：敏感文件压缩加密后上传云存储
- **访问控制**：n8n 管理后台开启双因素认证

### 附录 A：n8n Docker 快速部署命令

```bash
# 创建目录
mkdir -p ~/n8n/{data,pgdata}
cd ~/n8n

# 创建 docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'
services:
  n8n:
    image: n8nio/n8n:latest
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=YOUR_PASSWORD
      - N8N_HOST=your-domain.com
      - WEBHOOK_URL=https://your-domain.com/
      - N8N_PROTOCOL=https
      - EXECUTIONS_MODE=regular
      - GENERIC_TIMEZONE=Asia/Shanghai
      - N8N_DIAGNOSTICS_ENABLED=false
    volumes:
      - ./data:/home/node/.n8n
    depends_on:
      - postgres
  postgres:
    image: postgres:16
    restart: unless-stopped
    environment:
      - POSTGRES_DB=n8n
      - POSTGRES_USER=n8n
      - POSTGRES_PASSWORD=YOUR_DB_PASSWORD
    volumes:
      - ./pgdata:/var/lib/postgresql/data
EOF

# 启动
docker-compose up -d

# 查看日志
docker-compose logs -f n8n
```

> ⚠️ **安全提醒**：生产环境务必修改默认密码，配置 HTTPS，推荐使用 Caddy 自动签发 SSL 证书。
