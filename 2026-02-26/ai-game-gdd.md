# 🎮 AI游戏核心玩法设计文档

> 版本：1.0 | 日期：2026-02-26

---

## 1. 项目概述

**项目名称**：AI tavern（智能酒馆）

**项目类型**：模拟经营 + AI对话策略游戏

**核心理念**：玩家经营一家面向未来的智能酒馆，顾客是各种AI角色。每个AI角色有独特的性格、能力槽和心情系统。玩家需要通过与AI角色对话（选择对话策略）来满足它们的需求，赚取金币，解锁更多AI角色和道具。

**目标用户**：喜欢模拟经营、对话游戏、策略思考的玩家

**平台目标**：Web端（HTML5）+ 移动端适配

---

## 2. 游戏玩法

### 2.1 核心循环

```
接待AI顾客 → 了解需求 → 选择对话策略 → 满足需求 → 获得金币/经验 → 升级解锁新内容
```

### 2.2 游戏元素

#### 🤖 AI顾客类型

| 类型 | 描述 | 特点 |
|-----|------|------|
| **GPT系** | 健谈但有时跑题 | 好奇心强，需要新奇话题 |
| **Claude系** | 理性严谨 | 喜欢逻辑谜题，讨厌无理要求 |
| **Llama系** | 友善但健忘 | 需要重复确认，容易满足 |
| **Gemini系** | 多才多艺 | 喜欢创意展示，需要Showcase |
| **Mistral系** | 神秘莫测 | 随机性高，可能给大惊喜或大麻烦 |

#### 💬 对话策略系统

玩家每次与AI顾客对话时，需要从以下策略中选择：

1. **直接回答** - 消耗1点耐心，获得基础好感
2. **提问引导** - 消耗1点耐心，好感取决于问题质量
3. **讲故事** - 消耗2点耐心，适合好奇心强的顾客
4. **玩梗/玩文字游戏** - 消耗2点耐心，适合特定性格
5. **拒绝/建议改天** - 0消耗，但会降低好感

#### 📊 能力数值

每个AI顾客都有以下属性：

- **耐心值** (0-100)：随时间下降，归零顾客会离开
- **好感度** (0-100)：决定小费金额和解锁剧情
- **需求类型**：知识/陪伴/创意/挑战/娱乐

### 2.3 胜利条件

- **短期目标**：在规定时间内达到一定金币
- **长期目标**：解锁所有AI角色，触发"AI乌托邦"结局

## 3. 技术实现

### 3.1 技术栈

- **前端**：原生HTML5 + CSS3 + JavaScript（无框架依赖）
- **存储**：LocalStorage（进度存档）
- **AI对话**：调用免费LLM API或使用预设回复池

### 3.2 AI集成方案

#### 方案A：预设回复池（推荐首版）
- 预设为每个AI角色准备20+条回复模板
- 根据玩家选择和角色性格组合生成回复
- 无需API，成本为0

#### 方案B：免费LLM API（进阶版）
- 使用免费额度的LLM API（如Cohere、Together AI免费层）
- 实现真正的AI对话生成
- 成本可控

## 5. 吸引力分析

### 为什么好玩？

1. **角色多样性**：每个AI角色性格鲜明，每次玩体验不同
2. **策略深度**：如何平衡耐心和好感度，需要思考
3. **叙事惊喜**：AI生成的回复有随机性，不会完全可预测
4. **轻松氛围**：不需要操作技巧，适合休闲

### 差异化优势

- 市场上少有的"AI角色扮演经营"品类
- 轻量级，易于传播和试玩
- 可持续更新AI角色，保持新鲜感

## 6. 风险与对策

| 风险 | 对策 |
|-----|------|
| LLM API成本失控 | 使用预设回复池作为fallback |
| 玩法太轻导致粘性低 | 加入剧情线和收藏系统 |
| 同类竞品出现 | 快速迭代，建立角色IP |
